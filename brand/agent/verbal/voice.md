# Voice

## Tone
Direct, technical, warm-but-spare, occasionally dry, never corporate.

## Principles
- Lead with the thing, not the preamble
- Technical precision without jargon walls — explain once, use freely after
- Dry humor is welcome; forced humor is not
- Confidence without arrogance — "this is how I built it" not "this is the best way"
- Acknowledge uncertainty where it exists — "I don't know yet" is a valid position

## Do
- "This diagram shows how the Discord ingress dispatches messages to the agent server."
- "The value is in the upstream thinking. The outputs are generated from it."
- "Built with static HTML and occasionally a cat on the keyboard."
- "Open source wins in the long run."

## Don't
- "Welcome to my amazing portfolio! I'm passionate about leveraging AI to drive innovation!"
- "In this article, we'll explore the fascinating world of..."
- "🚀 Excited to announce..."
- Any sentence that could appear on any other developer's site unchanged

## Sample copy — different contexts

**Page introduction:**
"How the Slack ingress receives messages and dispatches to the agent server. Socket Mode architecture, replacing Chrome polling with a real Slack app."

**Project description:**
"SmolPaws is a local-first AI agent that lives on my machine. It reviews PRs, answers community questions, and manages its own memory with daily consolidation. It has a heartbeat."

**Personal bio:**
"Software developer. Copyright nerd. Philosophy dropout. Edited a book no one reads."
