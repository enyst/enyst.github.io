# Magic Trick

## The idea
The site is a living construction site, not a finished building. Every page has visible evidence of being built *by* an agent — because it is. The footer doesn't say "© 2026" — it says "Built with static HTML, Mermaid diagrams, and occasionally a cat on the keyboard." That's not a joke. That's the product.

## Why it works
Every developer site tries to look finished and polished. This one leans into the fact that it's an experimental notebook where an AI agent and a human are figuring things out together in public. The imperfection is the point. The "under construction" energy is authentic because things are *actually* under construction. Status tags on every card — `live`, `planned`, `data`, `report` — tell you honestly what state each thing is in.

## How to apply it
- Every new page should have a status tag: live / planned / draft / data / report
- The footer should always reference the actual tools used to build that page
- Never hide that SmolPaws contributed. The cat is a co-author, not a secret.
- Architecture diagrams should look like they were drawn on a whiteboard, not in a design tool — Mermaid, not Figma exports
- If something isn't done, say so on the page. Don't hide incomplete work — label it and ship it.

## References / inspiration
- Construction site fences with "FUTURE HOME OF..." signs — the promise is visible before the building exists
- Lab notebooks — dated entries, corrections visible, not cleaned up for publication
- The Whole Earth Catalog — "access to tools" energy, information-dense, self-published, opinionated
- Emmett Shine's observation: "the upstream definition work is becoming the actual product"
